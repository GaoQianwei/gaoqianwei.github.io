格雷编码

发表于 2021-05-26 分类于 算法 ， 数学 阅读次数： |

一、题目

    格雷编码是一个二进制数字系统，在该系统中，两个连续的数值仅有一个位数的差异。

    给定一个代表编码总位数的非负整数 n，打印其格雷编码序列。即使有多个不同答案，你也只需要返回其中一种。

格雷编码序列必须以 0 开头。

二、思路与代码

2.1 暴力解

    没有啥神奇的思路，只能一个个试，具体算法如下：

  1. 分析：相邻两个数的[汉明距离](汉明距离_百度百科 (baidu.com))为1，差值只能是 { 1，-1，2，-2，4，-4，8，-8，……，pow(2, n - 1)，-pow(2, n - 1) }；答案的元素是[ 1, pow(2, n - 1)]之间的值
  2. 设偏移数组 offset = { 1，-1，2，-2，4，-4，8，-8，……，pow(2, n - 1)，-pow(2, n - 1) }，布尔数组isUsed表示[ 1, pow(2, n - 1)]的某个元素是否已被用到。
  3. 确定结果的第一个元素为0，然后对下一个元素进行挑选（下一个元素即是当前元素加上某一偏移量且满足条件限制的情况）

2.2 leetcode题解

    这道题的核心其实在于了解格雷编码是如何形成的。

    第一个点，如果我们有 n bits，那么相应的我们会有 2^n^ 个格雷编码，假设 n=2，那么我们相应的会有 4 个格雷编码。所以我们先求出有多少个格雷编码也就是 2^n^ ，等价于 1 << n 也就是将 1 向左移动 n 位。假设n为2，那么0001向左移动两位就是0100也就变成了4.

    第二个点，格雷编码如何形成。同样以n = 2解析:

  • 公式为 (i ^ i) / 2 (i >> 1)
  • 0 ^ 0 = 00 ^ 00 = 00 = 0 计算(i/2): 0/2 = 0
  • 1 ^ 0 = 01 ^ 00 = 01 = 1 计算(i/2): 1/2 = 0
  • 2 ^ 1 = 10 ^ 01 = 11 = 3 计算(i/2): 2/2 = 1
  • 3 ^ 1 = 11 ^ 01 = 10 = 2 计算(i/2): 3/2 = 1
